## 01 — რა არის Vibe Coding

ტერმინი 2024 წელს მოიფიქრა **Andrej Karpathy**-მა (OpenAI-ის ერთ-ერთი დამფუძნებელი, ყოფილი Tesla AI Director). მისი თქმით:

> *„დაემორჩილე ვაიბებს, შეუშინდი exponentials-ს, და დაივიწყე, რომ კოდი არსებობს."*

ცხადი დეფინიცია: **Vibe Coding არის AI-ის დახმარებით პროდუქტის შექმნა ისე, რომ კოდის წერა შენ თვითონ არ გჭირდება**. შენ აღწერ რა გინდა — AI გიკეთებს.

შენი როლი:
- 🎯 **იდეის გადატანა მკაფიო ინსტრუქციებად** (prompt)
- 👀 **შედეგის შემოწმება** — მუშაობს თუ არა
- 🔄 **დახვეწა** — „ეს გადააკეთე ცოტა სხვანაირად"

ეს არ არის „AI კოდს ნელ-ნელა აგებინებს და შენ მერე გადაამოწმებ". ეს არის „AI აშენებს, შენ მართავ".

## 02 — ვისთვის არის

**Vibe Coding *არ* არის დეველოპერებისთვის.** დეველოპერი კოდს თვითონ წერს — და AI-ს იყენებს ხელშეწყობად.

Vibe Coding არის:

- 🚀 **სტარტაპერი**, ვისაც აქვს იდეა, მაგრამ MVP-ის ფული არ აქვს დეველოპერისთვის
- 📈 **მარკეტერი**, ვისაც ხშირად ჭირდება landing page, quiz, ან მინი-ხელსაწყო
- 🎨 **დიზაინერი**, ვისაც prototype-ი ცოცხალ აპად სურს გადააქციოს
- 💼 **ფრილანსერი**, ვისაც კლიენტისთვის სწრაფი deliverable-ი ჭირდება
- 🤔 **ცნობისმოყვარე**, ვისაც აქვს იდეა და უნდა ცადოს

**მთავარი — შენ უნდა იცოდე *რა* გინდა, არა *როგორ*.**

## 03 — ხელსაწყოები 2026-ში

### 🟢 Cursor
- **რა არის:** AI-პირველი IDE (Visual Studio Code-ის forked ვერსია)
- **ვისთვის:** ვისაც სურს ცოტა მაინც ჩახედოს კოდში
- **ფასი:** Free / $20-დან თვეში
- **ლინკი:** [cursor.com](https://cursor.com)

### 🟢 Claude Code
- **რა არის:** ბრძანების ხაზის ხელსაწყო (Terminal-ში მუშაობს)
- **ვისთვის:** ვისაც Terminal-ი არ აშინებს
- **ფასი:** Claude Pro/Max გამოწერით
- **ლინკი:** [claude.com/code](https://claude.com/code)

### 🟢 Lovable
- **რა არის:** ბრაუზერული აპის ბილდერი (chat-ით ცოცხალი აპი)
- **ვისთვის:** ნულოვანი technical background — ყველაზე მარტივი
- **ფასი:** Free trial / $20-დან თვეში
- **ლინკი:** [lovable.dev](https://lovable.dev)

### 🟢 v0 (Vercel)
- **რა არის:** UI/landing page გენერატორი
- **ვისთვის:** მარკეტერი, დიზაინერი, landing page-ისთვის
- **ფასი:** Free / $20-დან თვეში
- **ლინკი:** [v0.app](https://v0.app)

### 🟢 Bolt
- **რა არის:** Full-stack აპლიკაციის ბილდერი ბრაუზერში
- **ვისთვის:** SaaS-ის MVP, ფრილანსერი
- **ფასი:** Free / $20-დან თვეში
- **ლინკი:** [bolt.new](https://bolt.new)

### 🟢 Base44
- **რა არის:** No-code app builder, AI-ით გენერირებული
- **ვისთვის:** მეწარმე, ვისაც სრული შიდა აპი ჭირდება
- **ფასი:** Free / paid plans
- **ლინკი:** [base44.com](https://base44.com)

### 🟢 Replit Agent
- **რა არის:** Replit-ის AI agent რომელიც კოდს თვითონ წერს და run-ავს
- **ვისთვის:** Web-აპლიკაცია, prototype, scripting
- **ფასი:** Free tier / $20-დან თვეში
- **ლინკი:** [replit.com](https://replit.com)

> 💡 **არცერთი ცალკე არ არის „საუკეთესო".** მუშაობს stack-ი — სხვადასხვა ხელსაწყო სხვადასხვა task-ისთვის.

## 04 — პრაქტიკული Workflow — 4 ნაბიჯი

### ნაბიჯი 1 — დააფიქსირე იდეა მკაფიოდ

დაწერე 3-5 წინადადებაში:
- **რა გინდა აშენო?** (landing page, quiz, calculator, mini-app, ბოტი)
- **ვისთვის?** (target audience)
- **რა გადაწყვეტილებას აძლევს?** (the value)

მაგალითი:
> „მინდა landing page ჩემი ფიქტიური ყავის ბრენდისთვის. Target — 25-35 წლის თბილისელი specialty coffee mom-ი. გადაწყვეტა — single-origin მარცვალი + subscription model. ფერი: warm cream, accent terracotta."

### ნაბიჯი 2 — აირჩიე ხელსაწყო task-ის მიხედვით

| Task | ხელსაწყო |
|------|----------|
| Landing page | v0 ან Lovable |
| Full app (login + database) | Bolt ან Base44 |
| Telegram/Discord ბოტი | Claude Code |
| Prototype to test idea | Lovable |
| Custom tool / calculator | v0 ან Cursor |

### ნაბიჯი 3 — ცადე და გაამეორე

დაიწყე პირველი ვერსიით. ნუ ცდი ერთბაშად perfect იყოს.

- პირველი generation = 70% სავარაუდოდ
- მერე იტყვი: „ეს ცოტა გადააკეთე", „ფერი შეცვალე", „დაამატე ღილაკი", „გადააფასე layout"
- 3-5 iteration-ში მზად იქნება MVP

### ნაბიჯი 4 — გაუშვი

გადააქცი ცოცხალ პროდუქტად:
- **Lovable / Bolt / v0** — ერთი ღილაკით deploy
- **Cursor / Claude Code** — საჭიროა cloud hosting (Vercel, Netlify, Railway)

> 💡 ნაბიჯი 4-ის სიძნელე ხშირად ძირითადი blocker-ია. Lovable და v0 ამიტომ აღმოჩნდა ასე პოპულარული — deploy ერთი კლიკია.

## 05 — პირველი პროექტის იდეები

თუ ნულიდან იწყებ, აი 5 idea — ყველა vibe-coding-ით 1 დღეში დასრულდება:

### 1. ქართული ბრენდის Landing Page
შენი ან ფიქტიური ბრენდის სრული landing page — hero, features, testimonial, contact form.
**ხელსაწყო:** v0 ან Lovable
**დრო:** 2-3 საათი

### 2. Quiz თქვენი Audience-ისთვის
„რომელი ჩვენი პროდუქტი მოგიხდება?" 5-7 კითხვით — Instagram-ისთვის viral content.
**ხელსაწყო:** Lovable
**დრო:** 3-4 საათი

### 3. ROI Calculator შენი სერვისისთვის
ფრილანსერი ან consultant — კლიენტისთვის calculator („რამდენი დაგიზოგო თვეში?").
**ხელსაწყო:** v0
**დრო:** 2-3 საათი

### 4. Telegram Bot საკუთარი მიზნისთვის
Daily reminder, RSS aggregator, currency converter — რაც გაგვაგვერთებს.
**ხელსაწყო:** Claude Code
**დრო:** 4-6 საათი

### 5. Lead Magnet App
Free tool, რომელიც e-mail collection-ისთვის გამოიყენო — „გადათვალე შენი workflow", „დარეიტინგე შენი brand voice".
**ხელსაწყო:** Lovable ან Bolt
**დრო:** 4-6 საათი

## 06 — ხშირი შეცდომები

### ❌ შეცდომა 1: ბუნდოვანი იდეა

„მინდა აპლიკაცია სავარჯიშოებისთვის" → ცუდი prompt.
„მინდა Telegram bot, რომელიც ყოველდღე დილის 9-ზე გამიგზავნის 3 stretches-ის ვიდეო, dropdown menu-ით difficulty-ის ასარჩევად" → კარგი prompt.

### ❌ შეცდომა 2: პერფექციონიზმი პირველ ნაბიჯში

ცდილობ პირველი ვერსიამდე ყველაფერი იდეალური იყოს. ნუ აკეთებ ამას.
პირველი ვერსია — *მუშაობდეს*. დახვეწა მერეა.

### ❌ შეცდომა 3: ერთ ხელსაწყოზე ჩაკიდება

Cursor-ი არ მუშაობს რომელიმე task-ზე? სცადე Lovable. Lovable-ი limit-ში ჩავარდა? სცადე v0.
არცერთი ცალკე არ არის ყოვლისმოვალე.

### ❌ შეცდომა 4: Code-ის შეცვლა „გასწორებად"

თუ კოდი არ იცი — ნუ ცდი ხელით „გასწორებას". უთხარი AI-ს: „ეს ღილაკი არ მუშაობს, შეცვალე".

### ❌ შეცდომა 5: Deploy-ის გადადება

აშენე — გაუშვი იმავე დღეს. ცოცხალი ბმული, რომელიც სხვებისთვისაც გახსნადია, 100-ჯერ უფრო ღირებულია, ვიდრე ვერავის ნანახი ლოკალური ფაილი.

## 07 — შემდეგი ნაბიჯი

თუ ეს გზამკვლევი წაიკითხე და მზად ხარ ცადო, აი ჩემი რეკომენდაცია:

1. **დაიწყე Lovable-ით** — ყველაზე მარტივი, ბრაუზერში, არანაირი setup.
2. **აიღე 1 პროექტი 5 idea-დან** — დაიწყე უმარტივესით (landing page).
3. **გაუშვი 4 ნაბიჯში** — იდეა → ხელსაწყო → iteration → deploy.
4. **გააზიარე** — როცა ცოცხალ ბმულს მიიღებ, დადე LinkedIn-ზე ან Instagram-ზე. Feedback ცოცხალია — გაიგებ რა მუშაობს და რა არა.
