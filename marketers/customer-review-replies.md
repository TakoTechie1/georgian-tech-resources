## როგორ გამოვიყენო

1. იპოვე შაბლონი, რომელიც შენი review-ის ტიპს შეესაბამება
2. შეცვალე ფიგურულ ფრჩხილებში `[...]` მოცემული ცვლადები — `[სახელი]`, `[პროდუქტი]`, ა.შ.
3. დაამატე ერთი პერსონალური დეტალი — review-ის შინაარსიდან რომელიმე ფრაზა გამოიყენე
4. გადაამოწმე გრამატიკა და ბრენდის ხმა

> 💡 **მნიშვნელოვანი:** არასოდეს გამოიყენო შაბლონი word-by-word ყველა review-ისთვის. ალგორითმი (და ხალხი) გრძნობს copy-paste-ს.

## 01 — დადებითი reviews (5 შაბლონი)

### 1. მადლობა ზოგადად

**როდის:** კარგი review ფონს, კონკრეტიკის გარეშე

```
გამარჯობათ, [სახელი]! დიდი მადლობა თბილი სიტყვებისთვის — გუნდისთვის ეს ნამდვილი ენერგიის წყაროა. გელოდებით ისევ.
```

### 2. კონკრეტული სერვისი ან პროდუქტი

**როდის:** კლიენტი ასახელებს რა მოეწონა

```
[სახელი], მადლობა, რომ აღნიშნე [პროდუქტი/სერვისი]. ბევრი ფიქრი ჩავდევით ამ დეტალში — და მაგარია იცოდეთ, რომ ღირდა.
```

### 3. გრძელი დეტალური review

**როდის:** კლიენტი დიდი ცოცხალი ტექსტით წერს

```
[სახელი], დიდი მადლობა ცოცხალი feedback-სთვის. გუნდს გავუზიარებ — ეს მათი შრომის შედეგია. გელოდებით ისევ.
```

### 4. მეგობრებს გვირჩევს

**როდის:** კლიენტი წერს „ვირჩევ ყველას"

```
[სახელი], გული აგვიჩუყდა ნდობით — განსაკუთრებით მაშინ, როცა მეგობრებს გვირჩევთ. ეს ყველაზე ღირებული თანხმობაა.
```

### 5. 5 ვარსკვლავი ტექსტის გარეშე

**როდის:** მხოლოდ რეიტინგი, ტექსტი არაა

```
[სახელი], გმადლობთ 5 ვარსკვლავისთვის. ვიცით, რომ თქვენი დროის ერთი წუთიც კი მნიშვნელოვანია — ამიტომ ვაფასებთ.
```

## 02 — უარყოფითი reviews (5 შაბლონი)

### 1. სერვისი დაგვიანდა

**როდის:** კლიენტი წერს დროის პრობლემაზე

```
[სახელი], ვწუხვართ, რომ თქვენი გამოცდილება ჩვენს სტანდარტს არ შეესაბამებოდა. გთხოვთ, DM-ში მოგვწერეთ — გადასაჭრელად კონკრეტული ნაბიჯი დავგეგმოთ.
```

### 2. პროდუქტის ხარისხი

**როდის:** კლიენტი ჩივის პროდუქტზე

```
[სახელი], მადლობა, რომ დროულად აცნობეთ. ეს არ არის ხარისხი, რომელსაც ჩვენი ბრენდი ჰპირდება. DM-ში გამოგვიგზავნეთ შეკვეთის ნომერი.
```

### 3. თანამშრომელი უხეშობდა

**როდის:** კლიენტი წერს ცუდ მომსახურებაზე

```
[სახელი], გულწრფელად ვწუხვართ. ვერავინ უნდა იგრძნოს ცუდად ჩვენთან — ეს ჩვენი ერთი ღირებულებაა. DM-ში დაგვიკავშირდით — შიდად გავარჩევთ.
```

### 4. შეცდომით შეკვეთა

**როდის:** კლიენტმა არასწორი პროდუქტი მიიღო

```
[სახელი], ეს ჩვენი შეცდომაა და ბოდიში ვუხდით. DM-ში გამოგვიგზავნეთ შეკვეთის ნომერი — ხელახლა გამოგიგზავნით პროდუქტს ხარჯის გარეშე.
```

### 5. ვერ ვპოულობთ კლიენტს

**როდის:** review ცუდია, მაგრამ ჩვენს მონაცემებში არ ფიქსირდება

```
[სახელი], გასაგებია წუხილი, თუმცა მონაცემებში ვიზიტი ვერ ვიპოვე. შესაძლოა, შეცდომით სხვა გვერდი მოგექცათ. DM-ში მოგვწერეთ — გავარკვევთ.
```

## 03 — კითხვები review-ში (5 შაბლონი)

### 1. მუშაობის საათები

**როდის:** კლიენტი კითხულობს როდის ხართ ღია

```
[სახელი], მადლობა შეფასებისთვის. მუშაობის საათები: [HOURS]. წინასწარ ჯავშანი DM-ში ან [PHONE]-ზე. გელოდებით.
```

### 2. ფასები

**როდის:** კლიენტი კითხულობს რა ღირს

```
[სახელი], სრული ფასები ჩვენს website-ზე [LINK]. კონკრეტული სერვისი თუ გაინტერესებთ — DM-ში დაწერეთ, ცხადს გავხდი.
```

### 3. ხელმისაწვდომობა

**როდის:** კლიენტი ეძებს კონკრეტულ პროდუქტს

```
[სახელი], ეს პროდუქტი [STATUS — ხელმისაწვდომი/არაა/მოდის]. რეზერვაცია გვაქვს — DM-ში მოგვწერეთ კონტაქტი და შეგატყობინებთ.
```

### 4. ჩემს საჭიროებას ხომ უხდება?

**როდის:** კლიენტი დარწმუნდება საჭირო პროდუქტი თუ არის

```
[სახელი], კარგი კითხვაა. [PRODUCT] იდეალურია [USE CASE]-ისთვის. DM-ში მოგვწერეთ მცირე დეტალი — ზუსტ რეკომენდაციას მოგცემთ.
```

### 5. მიტანა / შიპინგი

**როდის:** კლიენტი კითხულობს მიწოდებაზე

```
[სახელი], მიტანა თბილისში [TIME] საათში, რეგიონებში [DAYS] დღეში. ფასი მისამართზე და შეკვეთის ღირებულებაზეა დამოკიდებული.
```

## 04 — ნეიტრალური / შერეული (5 შაბლონი)

### 1. 3 ვარსკვლავი — შერეული feedback

**როდის:** კლიენტი ნაწილობრივ კმაყოფილია

```
[სახელი], მადლობა გულწრფელი feedback-სთვის. [POINT]-ზე უკვე ვმუშაობთ. მომავალ ჯერზე გვინდა უკეთ მოვემზადოთ — გთხოვთ, კიდევ შანსი მოგვცეთ.
```

### 2. პირველი ვიზიტი — საშუალო

**როდის:** ახალი კლიენტი, არც ცუდი არც აღფრთოვანებული

```
[სახელი], მადლობა პირველი ვიზიტისთვის. ვხედავთ, რომ კარგი იყო, მაგრამ არა შესანიშნავი — სიგნალი მივიღეთ. შემდეგ ჯერზე 5 ვარსკვლავი ვცადოთ.
```

### 3. კონკურენტთან შედარება

**როდის:** კლიენტი წერს „X-ზე ცოტა უკეთესი/უარესი ხართ"

```
[სახელი], პატიოსანი შედარების მადლობა. ჩვენ ნამდვილად [DIFFERENT] — და ეს ყველას არ უხდება, რაც გასაგებია. გელოდებით თუ კი.
```

### 4. წინადადება ან რჩევა

**როდის:** კლიენტი წერს „გავაუმჯობესოთ X"

```
[სახელი], ფანტასტიკური წინადადებაა — გადავცემთ გუნდს. რასაც კლიენტი იძახის, ის ყველაზე სწრაფად გვაცეცხლებს ცვლილებას. მადლობა.
```

### 5. გრძელი დეტალური ნეიტრალური

**როდის:** კონკრეტული feedback-ით

```
[სახელი], დეტალური review-სთვის გულწრფელი მადლობა. ყველა ნოტიჩი დავიფიქსირეთ. ვცდილობთ, შემდეგი ვიზიტი 5 ვარსკვლავი იყოს.
```

## Best Practices — 5 ✓ და 5 ✗

### ✓ გააკეთე

1. **დაუძახე კლიენტს სახელით** — პერსონალიზაცია მუშაობს
2. **იყავი კონკრეტული, არა ზოგადი**
3. **უპასუხე 24 საათში** — ალგორითმი ეს იცის
4. **გადაიყვანე DM-ში თუ პრობლემაა** — საჯაროდ არ წყვიტო
5. **დაიჭირე ბრენდის ხმა** — ყველგან თანმიმდევრულობა

### ✗ არა

1. **არ გაიმეორო იგივე ტექსტი ყველა review-ზე**
2. **არ იყო defensive ცუდ review-ზე**
3. **არ უგულებელყო** — დუმილი ცუდი სიგნალია
4. **არ დაიწყო „ვწუხვართ"-ით** — ჯერ მადლობა, მერე სამუშაო
5. **არ დაუშვა AI-ის pure copy-paste** — ცოცხალი ენით

## 💡 Pro Tip — Brand Voice ერთხელ ჩაგდე AI-ში

თუ Claude-ს ან ChatGPT-ს იყენებ — შექმენი Project ან Custom GPT, სადაც ჩააგდებ:

1. შენი ბრენდის voice (ტონი, რა სიტყვებს იყენებ, რას არ იყენებ)
2. შენი 5 ყველაზე ხშირი review ტიპი
3. ერთი მაგალითი თითო ტიპში, რომელიც „იდეალურია" შენთვის

შემდეგ, ყოველ ჯერზე review-ის reply-ისთვის: **„გადააქციე ეს review შესაბამის reply-ად ჩვენი voice-ით"** — და მზადაა.
